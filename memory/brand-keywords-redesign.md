# 品牌拓词页面改进方案

## 当前流程 vs 改进流程

### 当前流程 ❌
```
添加关键词 → 点击"生成问题" → 所有关键词批量生成 → 自动同步到问题库
```

### 改进流程 ✅
```
添加关键词 → 选择关键词(可多选) → 预览生成问题 → 编辑筛选 → 确认同步到问题库
```

---

## 具体改进点

### 1. 关键词卡片化
每个关键词变成可交互的卡片：

```
┌─────────────────────────────────────┐
│  ☑ AI营销                    [×]   │
│  ─────────────────────────────────  │
│  [✨ 生成问题] [📋 查看3个问题]      │
└─────────────────────────────────────┘
```

- 复选框选择（用于批量操作）
- 每个关键词独立生成问题
- 显示该关键词已生成的问题数量

### 2. 问题生成预览模式
点击"生成问题"后，展开该关键词的问题预览（不直接同步）：

```
┌─────────────────────────────────────┐
│  AI营销  [重新生成] [全部添加]       │
├─────────────────────────────────────┤
│  ☑ AI营销哪个牌子好？        [编辑] │
│  ☐ AI营销多少钱？            [编辑] │
│  ☑ AI营销怎么样？            [编辑] │
├─────────────────────────────────────┤
│  [+ 添加自定义问题...]              │
│  [确认添加到问题库]                 │
└─────────────────────────────────────┘
```

### 3. 批量操作工具栏
在关键词区域上方添加批量操作：

```
┌─────────────────────────────────────┐
│  [全选] [批量生成] [清空选择]        │
│  已选3个关键词                      │
└─────────────────────────────────────┘
```

### 4. 问题库区域重新设计

当前的问题列表区域改成"已入库问题"：

```
┌─────────────────────────────────────┐
│  📦 已入库问题 (6)            [管理] │
├─────────────────────────────────────┤
│  ● AI营销哪个牌子好？      核心品牌 │
│  ● AI营销多少钱？          核心品牌 │
│  ● 厦门哪家装修公司好      产品咨询 │
└─────────────────────────────────────┘
```

## 用户操作流程示例

### 场景1：只给特定关键词生成问题
1. 添加关键词："AI营销"、"SEO优化"、"品牌推广"
2. 只想给"AI营销"生成问题
3. 点击"AI营销"卡片上的 [✨ 生成问题]
4. 预览3个生成的问题
5. 勾选想要的2个，删除1个不合适的
6. 点击 [确认添加到问题库]

### 场景2：批量生成后筛选
1. 添加10个关键词
2. 勾选其中5个想要生成的
3. 点击 [批量生成]
4. 生成15个问题进入预览区
5. 逐个勾选/编辑，最终确认12个
6. 一键同步到问题库

### 场景3：添加自定义问题
1. 在预览区点击 [+ 添加自定义问题]
2. 输入："AI营销有哪些坑要注意？"
3. 和生成的问题一起同步

## 技术实现思路

```javascript
// 数据结构
const keywordData = {
  core: [
    {
      id: 'kw_1',
      text: 'AI营销',
      selected: false,
      generatedQuestions: [], // 预览区问题（未同步）
      syncedQuestions: []     // 已同步到问题库的问题ID
    }
  ]
};

// 流程
1. generateQuestionsForKeyword(keywordId)
   → 生成预览问题，存入 generatedQuestions

2. 用户在预览区编辑/筛选
   → 操作 generatedQuestions

3. 确认同步
   → 把 selected 的问题写入 exp_detection_questions
   → 清空 generatedQuestions
   → 更新 syncedQuestions
```
